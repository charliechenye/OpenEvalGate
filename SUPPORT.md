# Support

OpenEvalGate is maintained as a public-alpha open-source project. Community support is best effort and there is no service-level agreement, guaranteed response time, or promise of production support.

## Where to ask

- Use a GitHub issue for a reproducible bug, documentation problem, or narrowly scoped feature proposal.
- Include the OpenEvalGate version, Python version, operating system, minimal synthetic or redacted inputs, and the command that reproduces the behavior.
- Use the issue forms when they fit the problem; they help keep security reports and ordinary support separate.

## What is in scope

Support can cover the local CLI, repository templates, documented schemas, deterministic validation behavior, report reproduction, and contribution workflow.

## What is out of scope

OpenEvalGate does not provide deployment approval, compliance certification, legal advice, security certification, runtime incident response, hosted execution, vendor-specific integration support, or assistance with private customer data. Do not post credentials, proprietary prompts, confidential policies, or unredacted evaluation traces.

Suspected vulnerabilities must be reported privately according to [SECURITY.md](SECURITY.md). The [Code of Conduct](CODE_OF_CONDUCT.md) applies to all project discussions.
