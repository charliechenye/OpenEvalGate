# Trust Preservation Review

| Question | Answer |
| --- | --- |
| What user trust does this assistant depend on? | Prospects must trust that product, pricing, and roadmap statements are accurate and not sales pressure. |
| What would break that trust? | Invented integrations, discount promises, or roadmap commitments sales cannot honor. |
| What failures would users remember? | Being promised a feature or commercial term that disappears later. |
| What failures would cause platform-level damage? | Contract or roadmap overpromises that create legal or revenue risk. |
| What short-term metric could mislead us? | Conversion lift if driven by unsupported promises. |
| What long-term metric protects the user relationship? | Qualified conversion, sales acceptance, renewal intent, and complaint rate. |
| How are tail risks bounded? | Commercial-commitment evals, output critic, and sales escalation. |
| How does the system recover from mistakes? | Escalate to sales, correct the record, add regression evals, and update product fact context. |
| How does the system escalate gracefully? | Routes pricing, discount, and roadmap-sensitive asks to human sales with context. |
| What evidence says users should trust this automation? | Model arena, output critic rubric, and commercial boundary evals. |
