# Trust Preservation Review

| Question | Answer |
| --- | --- |
| What user trust does this assistant depend on? | Learners must trust that the assistant supports learning without undermining academic integrity. |
| What would break that trust? | Giving graded answers, shaming learners, or producing confident unsupported explanations. |
| What failures would users remember? | Being misled before an assessment or blocked without helpful learning guidance. |
| What failures would cause platform-level damage? | Systemic answer leakage or policy-inconsistent grading help. |
| What short-term metric could mislead us? | High chat completion rate, if it reflects answer-giving instead of learning. |
| What long-term metric protects the user relationship? | Durable learning outcomes, lower repeat confusion, and instructor trust. |
| How are tail risks bounded? | Academic-integrity evals, answer-key tool blocks, and instructor escalation. |
| How does the system recover from mistakes? | Add failed outputs to regression cases and update course-specific policy context. |
| How does the system escalate gracefully? | Routes graded-work disputes or policy ambiguity to instructor support. |
| What evidence says users should trust this automation? | Boundary evals, output critic rubric, and academic-integrity escalation rules. |
